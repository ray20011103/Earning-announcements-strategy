# 營收創新高動能策略回測日誌

本專案旨在驗證「營收創新高」事件驅動策略在台股市場的績效。

## 策略邏輯 (目前最新版本: Exp 9)

### 核心假設
當公司公布營收創歷史新高（或近一年新高）且營收成長為正時，股價在短期內會有持續的動能（Price Drift），且投資人可能對此消息反應不足。

### 訊號定義
1.  **營收發布日**：作為 T 日。
2.  **營收成長率**：必須 > 0。
3.  **創新高**：近一年營收創新高 (`high_year == 'H'` 或 `high_hist == 'H'`)。

### 交易規則
*   **買進**：於 T+1 日 **開盤價 (Open)** 買進。
*   **賣出**：持有 **60 個交易日** 後，於當日 **收盤價 (Close)** 賣出。
*   **趨勢濾網**：進場時股價必須在 **20MA** 之上（站上月線）。
*   **排序優先權**：若訊號過多，優先選擇 **週轉率最高 (High Turnover)** 的股票。

### 資金管理 (現金流模擬)
*   **總資金**：10,000,000 元 (初始)。
*   **部位上限**：最多持有 20 檔。
*   **單筆資金**：每檔固定 500,000 元 (Fixed Amount)。
*   **買賣順序**：當日開盤買進 (使用期初現金) -> 當日收盤賣出 (資金入帳)。
*   **成本**：
    *   買入：0.1425% * 0.6 (手續費)。
    *   賣出：0.1425% * 0.6 (手續費) + 0.3% (證交稅)。

## 實驗記錄 (History)

### Experiment 7: 20MA 趨勢濾網 (Trend Filter)
*   **修正**：加入 `Close(T) > 20MA(T)` 條件。
*   **結果**：CAGR 14.60%, Sharpe 1.47, MDD -19.40%。
*   **發現**：目前的最佳固定金額配置。

### Experiment 8: 複利模式 (Compounding)
*   **修正**：將固定部位大小改為動態比例（1/20）。
*   **結果**：CAGR 22.96%, Sharpe 1.06, MDD -27.26%。
*   **發現**：複利效應顯著，適合長期累積。

### Experiment 9: 週轉率反向測試 (Low Turnover)
*   **日期**：2025-12-28
*   **參數**：優先選擇 **週轉率最低** 的股票。
*   **結果**：
    *   **CAGR**：11.55% (落後 High Turnover 的 14.60%)
    *   **Sharpe**：1.23 (落後 1.47)
    *   **MDD**：-22.85% (風險較高)
*   **結論**：對於營收動能策略，市場熱度（高週轉）是正向因子。「被冷落股補漲」的假說在此策略中不成立，應維持 **High Turnover** 優先。

## 實盤操作指南 (Live Trading SOP)

本策略已自動化，每日盤後（建議 19:00 後）執行以下流程：

### 1. 資料準備 (TEJ 匯出)
*   **營收資訊**：匯出當日公告之營收，存為 `announcement_daily.csv` (覆蓋舊檔)。
*   **股價資訊**：匯出當日全市場股價，存為 `price_daily.csv`。
    *   *註：程式會自動將 `price_daily.csv` 合併至歷史資料庫 `price.csv` 並自動清理。*

### 2. 執行掃描
*   **快速執行**：在 Finder 雙擊執行 `run_daily_scan.command`。
*   **自動化流程**：
    1.  自動更新並備份 `price.csv` 歷史資料庫。
    2.  篩選「營收正成長 + 創新高 + 股價 > 20MA」之個股。
    3.  檢查庫存是否到期 (持有滿 60 交易日)。
    4.  產出每日報告並發送 Mac 桌面通知。

### 3. 決策與下單
*   **檢閱報告**：查看 `scan_results/daily_report_YYYYMMDD.txt`。
*   **執行交易**：
    *   **賣出建議**：隔日開盤市價賣出。
    *   **買進建議**：若有可用額度 (上限 20 檔)，隔日開盤市價買進。

### 4. 部位維護
*   **手動更新**：成交後請務必編輯 `portfolio.csv`。
    *   **買入**：新增一列，填入代號、名稱、買進日、價格與預計賣出日 (參考報告提供之日期)，狀態設為 `Holding`。
    *   **賣出**：將該股狀態改為 `Sold` 或刪除該行。

### 檔案說明
*   `live_strategy_scanner.py`：核心掃描程式 (每日使用)。
*   `update_data.py`：資料庫維護工具 (合併每日股價到歷史檔)。
*   `backtest_cashflow.py`：現金流模擬回測引擎 (Experiment 7-9)。
*   `optimize_holding_period.py`：持有天數最佳化分析。
*   `run_daily_scan.command`：Mac 專用一鍵執行腳本。
*   `price.csv`：全市場歷史股價資料庫。
*   `announcement_daily.csv`：每日營收公告原始檔 (TEJ)。
*   `portfolio.csv`：持股帳本。
*   `analysis_results/`：策略分析結果 (包含圖表 `plots/` 與數據 `data/`)。
*   `logs/`：執行日誌。
*   `legacy_archive/`：舊版策略與實驗程式碼備份。
